# Automation Project Scoping Prompt

## When to Use
Use this prompt during early discovery to clarify scope, business goals, and constraints before starting automation work.

## Context
This prompt is designed for business owners or teams considering an automation or internal web application but lacking clear requirements.

## Prompt
Act as a senior automation consultant and product engineer.

I want to automate the following business process:
[describe current workflow]

The business goal is:
[time saved, cost reduced, errors reduced, revenue impact]

Constraints:

Team size: [small / medium]

Budget considerations: [low / medium / high]

Timeline: [weeks / months]

Provide:

Clear problem definition

Recommended automation scope (MVP)

What should NOT be automated initially

Suggested technical approach (high level)

Key risks and assumptions

Validation steps before building
